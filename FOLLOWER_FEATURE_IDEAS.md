# フォロワー一覧機能のアイデア

## パターン1: ダッシュボード統合型（シンプルカードリスト）
**概要**: ダッシュボードの既存カードと同じデザインで、フォロワー/フォロー中を表示

**特徴**:
- ダッシュボードの情報グリッドに追加
- カード形式で一覧表示
- フォロワー数とフォロー中数を表示
- 各ユーザーカードからプロフィールへ遷移可能
- フォロー/アンフォローボタン付き

**レイアウト**:
```
[プロフィールカード] [共有ツール]
[フォロワーカード]   [フォロー中カード]
```

**メリット**: 
- 既存デザインと統一感がある
- 実装が簡単
- 一目で状況が分かる

**デメリット**:
- フォロワーが多い場合、表示が限定的

---

## パターン2: 専用ページ型（タブ切り替え）
**概要**: `/dashboard/followers` という専用ページを作成

**特徴**:
- 「フォロワー」と「フォロー中」のタブで切り替え
- スクロール可能なリスト表示
- 検索・フィルター機能
- 各ユーザーにフォロー/アンフォローボタン
- プロフィール画像、名前、肩書を表示

**レイアウト**:
```
┌─────────────────────────────┐
│ [フォロワー] [フォロー中]   │ ← タブ
├─────────────────────────────┤
│ 👤 ユーザー1 [フォロー]     │
│ 👤 ユーザー2 [フォロー解除] │
│ 👤 ユーザー3 [フォロー]     │
│ ...                         │
└─────────────────────────────┘
```

**メリット**:
- 大量のフォロワーに対応可能
- 機能を拡張しやすい
- ユーザーが探しやすい

**デメリット**:
- ページ遷移が必要
- 実装コストがやや高い

---

## パターン3: モーダル/ドロワー型
**概要**: ダッシュボードのカードをクリックすると、モーダルまたはサイドドロワーで一覧を表示

**特徴**:
- ダッシュボードに「フォロワー: 12人」のようなカードを配置
- クリックでモーダル/ドロワーが開く
- モーダル内でタブ切り替え（フォロワー/フォロー中）
- スクロール可能なリスト
- 閉じるボタンで元の画面に戻る

**レイアウト**:
```
ダッシュボード:
[フォロワー: 12人] ← クリック

モーダル:
┌─────────────────────────┐
│ フォロワー一覧      [×] │
├─────────────────────────┤
│ [フォロワー] [フォロー中]│
│ 👤 ユーザー1            │
│ 👤 ユーザー2            │
│ ...                     │
└─────────────────────────┘
```

**メリット**:
- ページ遷移なしで閲覧可能
- モダンなUX
- スマホでも使いやすい

**デメリット**:
- モーダルの実装が必要
- 大量データの場合はパフォーマンス考慮が必要

---

## パターン4: プロフィールページ統合型
**概要**: プロフィールページ（`/[username]`）にフォロワー/フォロー中を表示

**特徴**:
- プロフィールページの下部にセクション追加
- 「12 フォロワー」「8 フォロー中」のようなカウント表示
- クリックで一覧を展開/折りたたみ
- グリッドまたはリスト表示
- 自分のプロフィールでも表示可能

**レイアウト**:
```
プロフィールページ:
┌─────────────────────┐
│ プロフィール情報     │
│ HamsaHand           │
│ 自己紹介            │
├─────────────────────┤
│ 12 フォロワー       │ ← クリックで展開
│ 8 フォロー中        │
│                     │
│ [フォロワー一覧]    │
│ 👤 👤 👤 👤        │
│ 👤 👤 👤 👤        │
└─────────────────────┘
```

**メリット**:
- プロフィール情報と一緒に見られる
- 他のSNSと同様のUX
- 自然な情報配置

**デメリット**:
- プロフィールページが長くなる可能性
- 実装がやや複雑

---

## パターン5: ハイブリッド型（推奨）
**概要**: ダッシュボードにサマリーカード + 専用ページで詳細表示

**特徴**:
- ダッシュボード: フォロワー数/フォロー中数を表示するカード
- カードをクリック → `/dashboard/followers` に遷移
- 専用ページでタブ切り替え（フォロワー/フォロー中）
- 検索機能、フィルター機能
- 各ユーザーにアクションボタン

**レイアウト**:
```
ダッシュボード:
[フォロワー: 12] [フォロー中: 8] ← クリック

/dashboard/followers:
┌─────────────────────────────┐
│ [フォロワー] [フォロー中]   │
├─────────────────────────────┤
│ 🔍 検索...                  │
│ 👤 ユーザー1 [フォロー]     │
│ 👤 ユーザー2 [解除]         │
│ ...                         │
└─────────────────────────────┘
```

**メリット**:
- ダッシュボードで概要を把握
- 詳細は専用ページで確認
- 拡張性が高い
- ユーザー体験が良い

**デメリット**:
- 実装コストがやや高い

---

## パターン6: インライン展開型
**概要**: ダッシュボードのカード内で展開/折りたたみ

**特徴**:
- ダッシュボードに「フォロワー」カードを配置
- 「もっと見る」ボタンでカード内に一覧を展開
- 折りたたみ可能
- 最初は3-5人だけ表示、展開で全員表示

**レイアウト**:
```
[フォロワー: 12人]
👤 ユーザー1
👤 ユーザー2
👤 ユーザー3
[もっと見る ▼]
```

**メリット**:
- ページ遷移なし
- シンプルな実装
- コンパクト

**デメリット**:
- 大量データには不向き
- カードが長くなる可能性

---

## 推奨実装順序

1. **パターン5（ハイブリッド型）** - 最もバランスが良い
2. **パターン2（専用ページ型）** - シンプルで実装しやすい
3. **パターン3（モーダル型）** - モダンなUX

## 実装に必要な要素

### API エンドポイント
- `GET /api/followers` - フォロワー一覧取得
- `GET /api/following` - フォロー中一覧取得

### Server Actions
- `getFollowers(userId)` - フォロワー取得
- `getFollowing(userId)` - フォロー中取得

### コンポーネント
- `FollowersList.tsx` - フォロワー一覧表示
- `FollowingList.tsx` - フォロー中一覧表示
- `FollowerCard.tsx` - 個別ユーザーカード

### ページ
- `/dashboard/followers/page.tsx` - フォロワー一覧ページ（パターン2,5の場合）
